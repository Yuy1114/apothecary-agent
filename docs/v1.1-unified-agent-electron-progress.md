# Apothecary Agent v1.1：统一 Agent 与 Electron 入口

> 状态日期：2026-07-03  
> 基线：[`v1-product-boundary-and-roadmap.md`](./v1-product-boundary-and-roadmap.md)  
> 前置版本：[`v1-completion-report.md`](./v1-completion-report.md)

## 结论

v1.1 已完成首个可独立运行的桌面闭环。用户不再必须通过 Mastra Studio 使用 Apothecary；Electron App 已成为产品入口，Mastra Studio 保留为开发与调试工具。

## 已完成内容

### 1. 统一 Agent

- 新增 `Apothecary` 总 Agent，统一承接问答、知识沉淀、Inbox 整理、变更处理、知识画像和维护建议；
- 用户不需要选择内部专职 Agent；
- 所有人类可读文件修改仍只能生成 proposal；
- 对话入口不能直接批准 proposal，批准与拒绝必须在桌面端提案页完成。

### 2. Electron 产品入口

桌面端已经提供五个一级入口：

| 入口 | 能力 |
| --- | --- |
| 对话 | 与总 Agent 对话，提出检索、整理和沉淀意图 |
| 变更 | 查看待处理文件，读取当前内容，标记处理或忽略，手动同步 |
| Inbox | 查看待归位文档，并把内容交给 Agent 生成移动提案 |
| 提案 | 查看 proposal 详情，显式批准或拒绝 |
| 知识画像 | 查看 profile、关系、候选项和维护发现 |

### 3. 桌面安全边界

- Renderer 不启用 Node.js；
- Preload 只暴露白名单 IPC；
- `contextIsolation` 与 sandbox 均开启；
- 文件写入继续复用 v1 proposal、路径安全、operation ledger 与 recovery 机制；
- 首次启动在用户数据目录创建独立数据库，不依赖项目目录可写。

### 4. 首次启动与 Vault 选择

- 启动时按环境变量、已保存选择、兼容默认路径的顺序寻找 Vault；
- 没有可用路径时显示 macOS 原生目录选择器；
- 选择结果保存在 Electron 用户数据目录；
- 可从“药柜 → 选择其他药柜…”切换，App 会重启并让所有 Agent 工具加载同一个新路径。

### 5. 系统状态诊断

- 桌面端新增“系统状态”入口；
- 分别检测 DeepSeek 对话模型、Embedding 服务和本地 Vault 权限；
- 服务检测会验证 `/models` 网络连接与鉴权，并区分缺少 Key、鉴权失败、服务错误和网络不可达；
- API Key 只在主进程使用，Renderer 只能收到脱敏后的状态、模型名和服务主机。

### 6. React Renderer

- Electron Renderer 已从原生 HTML/JavaScript 事件驱动实现迁移为 React；
- 对话、变更、Inbox、提案、知识画像和系统状态均拆分为独立视图组件；
- 保留现有 IPC 合约、安全边界、视觉样式和跨页面 Agent prompt 行为；
- Vite 构建和 Renderer TypeScript 检查已纳入统一 `build` / `check` 命令。

### 7. 单窗口 Agent Run

- 对话入口改为消费 Mastra `fullStream`，保留文本增量、tool call、tool result 和错误事件；
- Electron IPC 将一次执行建模为带 `runId` 的事件流；
- React 对话时间线会原位展示工具执行状态和本次 Run 新生成的 proposal；
- proposal 可在时间线内批准或拒绝，处理后自动发起同线程 follow-up，总结最终状态；
- 当前 follow-up 是连续对话 Run，后续再升级为 Mastra 原生 suspension/resume。

## 使用方式

```bash
pnpm install
pnpm run desktop:dev
```

生成本机 App：

```bash
pnpm run desktop:pack
open release/mac-arm64/Apothecary.app
```

仍需通过 `.env` 提供模型和 embedding 所需的密钥，并通过 `APOTHECARY_VAULT_PATH` 指定 Vault；未指定时沿用项目默认 Vault。

## 当前发行边界

- 当前构建面向 macOS arm64；
- App 尚未配置正式图标和 Apple Developer ID 签名；
- 为兼容 pnpm 别名依赖，首版关闭 asar 并显式复制运行时别名包；后续可改为专用 bundling 流程以缩小体积；
- 单次运行只连接一个本地 Vault，但可通过应用菜单切换；
- 对话为本次启动内的会话，尚未提供桌面端历史会话管理。

## 下一阶段建议

1. 将 proposal follow-up 升级为原生 suspend/resume，并加入取消能力；
2. 配置应用图标、签名、公证与自动更新；
3. 为 React Renderer 增加端到端交互测试。
